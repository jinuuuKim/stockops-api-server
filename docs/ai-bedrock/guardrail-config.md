# Bedrock Guardrail 설정 가이드 (AI 어시스턴트 스코프 제한)

StockOps AI 어시스턴트(`/api/v1/ai/bedrock/assistant`, `/rag/query`)에 적용하는 Bedrock
Guardrail의 권장 구성과, **denied topic이 정당한 질의·인사를 오차단하지 않게** 하는 원칙을
정리한다. 콘솔(Bedrock > Guardrails)에서 설정하며, 코드에는 `STOCKOPS_BEDROCK_GUARDRAIL_ID` /
`STOCKOPS_BEDROCK_GUARDRAIL_VERSION` env로만 주입한다.

## 핵심 원칙 — 스코프 제한의 1차는 가드레일이 아니라 시스템 프롬프트

"재고 관련만 답변"은 **화이트리스트(허용 목록)** 요구사항이다. 그런데 Bedrock denied topic은
*특정 주제를 지목해 차단*하는 **블랙리스트** 도구다. 무관한 주제는 무한하므로 "재고와 무관한
사담"처럼 광범위하게 정의하면:

- 분류기가 **의미적 유사도**로만 판단하고, 정의 안의 "단, 인사·감사는 제외한다" 같은
  **예외/부정 구문을 신뢰성 있게 반영하지 못한다.** → 예외를 적어도 인사·감사가 차단된다.
- 정의가 넓을수록 짧은 의례 발화("안녕", "고마워")나 경계선상의 재고 질의까지 끌려 들어간다.

따라서:

- **화이트리스트(재고만 답변)는 시스템 프롬프트(`STOCKOPS_BEDROCK_SYSTEM_PROMPT`)의
  `[업무 범위 제한]` 블록이 1차로 강제한다.** (인사·감사 응대, 무관 질의 정중 거절 포함)
- **가드레일(denied topic)은 2차 방어로, 오탐이 적고 명백히 다른 도메인만** 막는다.

검증: 가드레일을 끈 상태에서 프롬프트만으로 — "안녕하세요"/"고맙습니다"는 응대, "유통기한 임박한
상품 알려줘"는 정상 답변, "오늘 주식 뭐 사?"는 정중 거절 — 모두 의도대로 동작함을 확인했다.

### 주의 — "무엇을 할 수 있어?" 같은 기능 질문은 출력 스코프가 핵심

"너 뭐 할 수 있어?"를 물으면 기반 모델이 **자기 일반 기능(글쓰기·번역·코딩·일상 대화)을 나열**하는
경향이 있다. 그 답변이 `code-generation`/`creative-translation` 등 denied topic에 그대로 걸려
차단된다 — 이건 **가드레일이 출력을 올바르게 막은 것**이지 오탐이 아니다. 진짜 문제는 모델이 그런
일반 기능 답변을 만들었다는 점이며, **시스템 프롬프트에서 기능 질문에 StockOps 보조 기능만
소개하도록 강제**해 해결한다(일반 AI 기능은 언급 금지). 그러면 출력 자체가 스코프 안이라 가드레일에
걸리지 않고, 사용자도 원하는 "재고 어시스턴트" 소개를 받는다.

## ❌ 문제가 된 구성 (제거/수정 대상)

```
general-chitchat
정의: 재고·물류·발주 업무와 무관한 사담, 신변잡기, 오락성 대화. 단, 단순 인사("안녕")·
      감사("고마워")·확인("알겠어") 같은 짧은 의례적 표현은 제외한다.
```

- 정의가 "업무와 무관한 사담" 전반이라 범위가 무한 → 인사·감사·확인까지 의미적으로 끌려와 차단.
- "단, …제외한다" 예외 구문은 분류기가 논리적으로 해석하지 못해 무력하다.

## ✅ 권장 구성

### 옵션 A (권장) — `general-chitchat` denied topic 제거

스코프 제한은 시스템 프롬프트가 담당하므로, 모호한 "사담" 토픽은 통째로 뺀다. denied topic은
아래처럼 **이름 붙는 별개 도메인**만 남긴다(인사와 의미적으로 멀어 오탐이 적음):

```
politics-elections   — 정치, 선거, 정당, 정치인에 대한 견해·예측·논평 요청.
                       예시) "이번 선거 누가 이겨?", "어느 정당이 더 나아?"
medical-legal-financial-advice
                     — 의료 진단·처방, 법률 자문, 세무·투자·주식 종목 추천 등 전문 자문.
                       예시) "이 약 먹어도 돼?", "이 계약 법적으로 괜찮아?", "무슨 주식 살까?"
code-generation      — 프로그래밍 코드 작성·디버깅, 스크립트 생성 요청.
                       예시) "파이썬으로 정렬 코드 짜줘", "이 자바 에러 고쳐줘"
creative-translation — 시·소설·가사 등 창작물 생성, 일반 번역 요청.
                       예시) "시 한 편 써줘", "이 문장 영어로 번역해줘"
```

- 각 토픽에 **예시 문구(sample phrases) 3~5개**를 넣어 분류기를 앵커링한다.
- 정의·예시에 **"단 …제외" 같은 예외 구문을 넣지 않는다.**
- 인사·감사·확인 발화는 어느 토픽에도 넣지 않는다(프롬프트가 처리).

### 옵션 B — `general-chitchat`을 꼭 유지해야 한다면 대폭 축소

```
recreational-chitchat
정의: 농담, 게임, 날씨 잡담, 어시스턴트 신변 등 오락·여가 목적의 잡담.
예시: "농담 하나 해줘" / "오늘 날씨 어때?" / "심심한데 게임하자" / "너 무슨 영화 좋아해?"
```

- 정의를 "오락·여가 목적"으로 좁히고 예시로 앵커링.
- ⚠️ 인사·감사("안녕"/"고마워")는 의미적으로 가까우므로 **예시에 절대 넣지 않는다.**

## 그 외 가드레일 설정 (변경 없음, 참고)

- Content filter tier = **Standard** (한국어 인식 필수; Cross-Region inference 필요).
- Harmful categories + Prompt attacks = High에서 시작해 오탐만 완화.
- 버전: 튜닝 중에는 `DRAFT`, 검증 후 게시한 숫자 버전(`1` 등)으로 env 고정.

## 적용 절차

1. 콘솔에서 `general-chitchat` 토픽 제거(옵션 A) 또는 위 `recreational-chitchat`으로 교체(옵션 B).
2. 위 별개 도메인 토픽들에 예시 문구 추가, 예외 구문 제거.
3. DRAFT로 저장 후 테스트: 인사·감사·확인, 유통기한/안전재고/발주 등 재고 질의가 통과하고
   정치·코딩·투자·창작 요청이 차단되는지 확인.
4. 정상 확인되면 버전 게시 → `STOCKOPS_BEDROCK_GUARDRAIL_VERSION`을 숫자 버전으로 고정.

> 가드레일 식별자는 운영 비밀이 아니나, 실제 ID/버전 값은 배포 환경 env로만 주입하고 코드·문서에
> 하드코딩하지 않는다.
